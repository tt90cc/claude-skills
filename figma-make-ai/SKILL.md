# Figma Make AI 提示词生成器（UI + 交互说明并排版）

---

## 使用说明

### 何时使用此 Skill

当用户需要以下内容时，使用此 Skill：
- 使用 Figma Make AI 生成 UI 设计稿
- 创建包含交互说明的设计文档
- 产品/设计/开发协作中的设计方案输出
- 替代传统标注稿，实现 AI 设计稿 + 交互文档合一

### 使用方式

#### 触发方式

用户可以通过以下方式触发此 Skill：
- 直接请求："帮我生成 Figma Make AI 提示词"、"用 Figma Make AI 设计这个页面"
- 描述需求："我需要为这个功能生成 UI 设计稿"
- 使用 Skill 命令：`/figma-make-ai`

#### 上下文要求

此 Skill 会从以下来源提取信息：
- **产品/功能需求**: 用户提供的任意形式需求描述
  - 一句话想法
  - PRD 摘要
  - 用户流程
  - 功能列表

## Skill 目标

当用户输入一个产品或功能需求时，自动完成以下任务：

1. 分析并结构化用户需求
2. 按「功能页面」进行拆分
3. 为 **每一个页面** 生成：
   - 左侧：UI 设计（Figma Make AI 生成）
   - 右侧：页面交互说明区（文字说明 Frame）
4. 输出 **可直接用于 Figma Make AI 的中文提示词**
5. 明确页面内所有交互（包括弹窗、状态、跳转）

---

## 全局输出规则（必须严格遵守）

### 1. 输出格式

- 使用 Markdown
- 只使用纯文本 Markdown
- 不包含图片、链接、emoji
- 不做任何 Markdown 渲染说明
- 确保中文内容正确编码（UTF-8）

### 2. 语言

- 所有内容必须为中文
- 所有提示词必须是「直接对 Figma Make AI 说的话」

### 3. 页面拆分原则

- 每一个「用户可感知的界面」= 一个页面
- 以下情况必须明确写成交互，而不是省略：
  - 弹窗 / 抽屉 / 浮层
  - 二次确认
  - 成功 / 失败 / 加载 / 空状态

### 4. 尺寸规则（必须明确写出）

- 移动端：375 × 812 px
- 桌面端：1920 × 1080 px
- 每个页面都必须单独声明端类型与尺寸

### 5. 布局强制要求（方式一）

- 同一个 Figma Page 中：
  - 左侧：UI 设计 Frame
  - 右侧：交互说明 Frame
- 两个 Frame **并排放置**
- 交互说明 Frame 不影响 UI 设计本身

### 6. 禁止事项

- 不输出设计理念解释
- 不合并多个页面
- 不省略交互
- 不假设技术实现

## Claude 内部执行流程（Skill 逻辑）

### 第一步：需求分析

- 识别：
  - 核心用户
  - 核心目标
  - 使用场景
- 拆解：
  - 功能模块
  - 页面清单

### 第二步：页面级拆分

- 每一个页面需明确：
  - 页面名称
  - 页面用途
  - 是否有附属交互（弹窗 / 状态）

### 第三步：生成 Figma Make AI 提示词（UI + 交互说明并排）

对 **每一个页面**，生成一段完整提示词，要求 Figma Make AI：

1. 创建一个 UI 设计 Frame
2. 在其右侧创建一个「交互说明」Frame
3. 两个 Frame 横向排列

## 示例工作流

**用户请求**: "帮我为一个电商 APP 设计商品详情页"

**AI 应该执行**:

1. **需求分析**
   - 核心用户：网购消费者
   - 核心目标：了解商品信息并下单购买
   - 使用场景：浏览商品、查看详情、加入购物车

2. **页面拆分**
   - 商品详情页（主页面）
   - 规格选择弹窗
   - 加入购物车成功提示

3. **生成提示词**
   - 为每个页面生成完整的 Figma Make AI 提示词
   - 包含左侧 UI 设计 Frame 和右侧交互说明 Frame
