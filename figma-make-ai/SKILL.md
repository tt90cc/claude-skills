# Figma Make AI 提示词生成器 v2.0

## 概述

本技能用于生成清晰、结构化的 Figma Make AI 提示词，帮助用户快速将产品需求转化为可视化设计。

**核心特性：**
- 多页面并排生成，使用绝对坐标定位（x、y 坐标），确保精确排布
- 支持移动端/桌面端预设尺寸（375×812px / 1280×800px）
- 高度规则：内容不超一屏时固定高度，超出时向下延伸
- 支持用户自定义尺寸
- 详细的元素描述和交互说明
- 确保设计一致性和交互状态完整性
- 纯中文输出，适配 Figma Make AI

**适用场景：**
- 产品需求快速可视化
- 交互原型设计
- 设计评审准备
- 开发协作文档

**不适用场景：**
- 高保真视觉设计（需设计师手工调整）
- 复杂交互动画（需原型工具）
- 品牌视觉设计（需品牌指南）

## 全局输出规则（必须严格遵守）

### 1. 输出格式

- 使用 Markdown
- 只使用纯文本 Markdown
- 不包含图片、链接、表格、emoji
- 不做任何 Markdown 渲染说明
- 确保中文内容正确编码（UTF-8）

### 2. 语言

- 所有内容必须为中文
- 所有提示词必须是「直接对 Figma Make AI 说的话」

### 3. 页面定义原则

**多页面并排生成策略：**
一次性生成多个独立页面，使用绝对坐标定位在同一个 Figma 画布上并排排列。

**什么是一个「页面」：**
- 用户可感知的一个完整界面状态
- 每个交互状态都是一个独立的页面
- 包括但不限于：
  - 主功能页面
  - 弹窗/抽屉/对话框（作为独立的完整页面）
  - 确认对话框
  - 成功/失败/错误提示页面
  - 加载状态页面
  - 空状态页面

**生成流程：**
1. 理解用户需求，识别所有需要设计的页面
2. 列出完整的页面清单，确认范围
3. 为所有页面生成一个合并的 Figma Make AI 提示词
4. 在提示词中使用绝对坐标（x、y）定位每个页面，确保并排排列

### 4. 尺寸规范

**默认预设尺寸：**

**移动端：**
- 宽度：375 px（固定，不得自动调整）
- 高度：812 px（内容不超一屏时固定；超出一屏时向下延伸）

**桌面端：**
- 宽度：1280 px（固定，不得自动调整）
- 高度：800 px（内容不超一屏时固定；超出一屏时向下延伸）

**自定义尺寸支持：**
- 如果用户在对话中明确提供了特定尺寸（如 "设计一个 414×896 的页面"），则使用用户指定的尺寸
- 自定义尺寸格式：[宽度] × [高度] px
- 必须在提示词中明确标注实际使用的尺寸

**尺寸选择逻辑：**
1. 优先使用用户明确提供的尺寸
2. 如果用户只指定设备类型（移动端/桌面端）但未提供尺寸，使用对应的默认预设
3. 如果用户未指定任何尺寸信息，根据需求场景选择合适的预设

### 5. 禁止事项

- 不输出设计理念解释
- 不省略交互
- 不假设技术实现
- 不使用相对宽度（必须使用绝对像素值）
- 不忽略强制约束声明

   - 确认页面范围和优先级

### 第二阶段：生成合并提示词

   - 详细描述每个页面的布局和元素
   - 包含完整的交互说明和页面关联

### 第三阶段：验证与调整

7. **验证生成结果**
   - 确认生成的页面在 Figma 上正确并排排列
   - 如需调整，修改坐标或约束声明后重新生成

## 尺寸处理流程

在生成 Figma Make AI 提示词之前，必须先确定页面尺寸：

1. **检查用户输入**
   - 用户是否提供了明确的尺寸数字？（如 "1440×900"、"375×667"）
   - 用户是否提到了特定设备型号？（如 "iPhone 14"、"iPad Pro"）
   - 用户是否指定了设备类型？（如 "移动端"、"桌面端"）

2. **确定尺寸**
   - 如有明确尺寸 → 直接使用
   - 如有设备类型但无具体尺寸 → 使用对应预设（移动端 375×812，桌面端 1280×800）
   - 如无任何指示 → 根据产品功能特性判断更适合的设备类型，使用对应预设

3. **在提示词中明确标注**
   - 必须在生成的提示词开头明确标注使用的尺寸和高度规则
   - 格式：**尺寸：** [宽度] × [高度] px（内容不超一屏时固定；超出时向下延伸）
   - 对于预设尺寸，必须强调宽度固定不变，高度根据内容灵活调整

---

## 强制约束声明规范

为确保 Figma Make AI 严格遵守尺寸规范，必须在提示词开头添加强制约束声明：

**格式示例：**
```
【全局强制约束】
- 每个页面画板宽度严格等于 [宽度]px，不得自动调整
- 高度规则：内容不超出一屏时高度固定为 [高度]px；内容超出一屏时仅向下延伸，宽度仍固定 [宽度]px
- 所有元素宽度不得超过 [宽度]px
- [其他特定约束]
```

**关键要点：**
- 使用"严格等于"而非"约等于"或"大约"
- 明确说明宽度固定不变
- 明确说明高度的两种情况（固定 vs. 延伸）
- 对于多页面，明确列出每个页面的坐标位置

**多页面坐标定位示例：**
```
**页面坐标：**
- 页面1：x=0，y=0，宽=375px，高=812px（内容不超一屏，固定高度）
- 页面2：x=415px，y=0，宽=375px，高=812px（内容不超一屏，固定高度）
- 页面3：x=830px，y=0，宽=375px，高度随内容延伸
```

**坐标计算方法：**
- 第一个页面：x=0
- 后续页面：x = 前一页面 x + 前一页面宽度 + 间距（通常 40px）
- 所有页面：y=0（从顶部开始）
- 高度：固定页面用 812px（移动端）或 800px（桌面端），列表页面用"随内容延伸"

## 多页面中的交互上下文说明

当生成多个页面的提示词时，需要为每个页面简要说明：
- **来源：** 用户从哪里进入这个页面（如 "从主页点击删除按钮后显示"）
- **目的：** 这个页面要实现什么功能
- **后续：** 用户操作后会发生什么（如 "点击确认后显示成功页面"）
- **页面关联：** 此页面与其他页面的关系（如 "与订单列表页并排显示"）

**示例：**
```
**交互说明：**
- 来源：用户在订单列表页点击某个订单的删除图标后显示此确认页面
- 目的：让用户二次确认删除操作，避免误删
- 后续：点击"确认删除"按钮后将显示删除成功页面；点击"取消"按钮返回订单列表页
```

## 元素描述最佳实践

**清晰的元素描述应包含：**

1. **类型明确**
   - ✓ "主操作按钮"
   - ✗ "一个东西"

2. **位置准确**
   - ✓ "位于页面顶部，距离顶边 20px"
   - ✗ "在上面"

3. **内容完整**
   - ✓ "标题文本：'确认删除订单'，字号 18px，粗体，深灰色 #333333"
   - ✗ "有个标题"

4. **样式具体**
   - ✓ "红色背景 #FF4D4F，白色文字，圆角 8px，高度 48px，宽度占满屏幕左右各留 16px 边距"
   - ✗ "红色按钮"

5. **层次清晰**
   - 使用缩进和编号表示元素的嵌套关系
   - 先描述容器，再描述容器内的元素

6. **坐标定位准确**（多页面必须）
   - ✓ "位于 x=0，y=0，宽=375px，高=812px"
   - ✓ "距离顶部 20px，距离左边 16px，宽度 343px"
   - ✗ "在左上角"
   - ✗ "靠左对齐"
   - 使用绝对坐标而非相对位置
   - 明确参考点（屏幕边缘、父容器边缘）
   - 提供完整的位置信息（x、y、宽、高）

## 交互状态完整性检查

**生成提示词前必须确认的状态：**

**基础交互状态：**
- [ ] 初始/默认状态（页面首次加载）
- [ ] 加载中状态（数据请求中）
- [ ] 成功状态（操作成功）
- [ ] 错误状态（操作失败/网络错误）
- [ ] 空状态（无数据/无内容）

**表单相关状态：**
- [ ] 输入框聚焦/失焦状态
- [ ] 输入验证失败状态（错误提示）
- [ ] 按钮禁用/可用状态
- [ ] 必填项标识

**确认类交互状态：**
- [ ] 二次确认页面（删除、提交等重要操作）
- [ ] 确认后的成功/失败反馈

**用户引导状态：**
- [ ] 首次使用引导页面
- [ ] 权限请求页面
- [ ] 功能说明页面

**在提示词的交互说明部分，必须明确列出该页面涵盖的状态。**

## 状态转换描述规范

**每个页面的交互说明必须包含：**

1. **触发条件**
   - 从哪个页面/状态来
   - 通过什么操作触发（点击、输入、滑动等）
   - 是否有前置条件（登录、权限等）

2. **当前状态行为**
   - 页面显示什么内容
   - 用户可以进行哪些操作
   - 是否有自动行为（自动跳转、倒计时等）

3. **后续转换**
   - 每个可操作元素会导向什么结果
   - 是否跳转到其他页面
   - 是否改变当前页面状态
   - 是否调用系统功能（分享、相机等）

**示例：**
```
**交互说明：**

**触发条件：**
- 来源页面：订单列表页
- 触发操作：点击订单项右侧的删除图标
- 前置条件：用户已登录且有删除权限

**当前页面行为：**
- 以弹窗形式覆盖在订单列表页上方，背景半透明遮罩
- 显示待删除订单的关键信息（订单号、金额）
- 提供"取消"和"确认删除"两个操作按钮
- 无自动行为，等待用户选择

**后续转换：**
1. 点击"取消"按钮
   - 结果：关闭确认弹窗，返回订单列表页
   - 数据：不进行任何数据操作

2. 点击"确认删除"按钮
   - 结果：关闭当前弹窗，显示"删除中"加载状态，调用删除 API
   - 成功：跳转到删除成功页面
   - 失败：跳转到删除失败页面，显示错误原因

3. 点击背景遮罩
   - 结果：与点击"取消"按钮相同
```

```
**辅助状态页面：**
```

- 全局强制约束声明

- 每个页面的详细描述

## 多页面合并提示词模板

**完整的多页面合并提示词结构示例：**

```
请在同一个 Figma 画布上，一次性生成以下 3 个移动端页面，从左到右并排排列。
```

### 常见问题解决

**问题 1：Figma Make AI 生成的页面宽度不是 375px，而是更大的尺寸（如 2000px）**

解决方案：
- 在提示词开头添加强制约束声明：`【强制约束】画板宽度严格等于 375px，不得自动调整`
- 在所有元素描述中使用具体的 px 数值，避免使用"100%"等相对宽度
- 明确标注每个元素的具体宽度（如"宽度 343px = 375 - 左右各 16px 边距"）
- 避免模糊的宽度描述，确保 Figma Make AI 无法自由发挥

**问题 2：Figma Make AI 生成的页面高度远超 812px，即使内容不多**

解决方案：
- 在提示词中明确说明高度规则：`高度规则：内容不超一屏时高度固定为 812px；内容超出一屏时仅向下延伸`
- 在布局结构中明确各区块的高度分配，确保总高度等于 812px
- 对于列表页等可能超出的页面，明确标注"高度随内容延伸，最小高度 812px"
- 避免使用"自适应高度"等模糊表述

**问题 3：Figma Make AI 生成的元素位置不准确**

解决方案：
- 使用"距离 XX 边 XX px"而不是"位于 XX 位置"
- 明确参考点（屏幕边缘、父容器边缘、其他元素）
- 提供多个位置描述（如"距顶部 20px，距左边 16px"）

**问题 4：生成的页面元素缺失**

解决方案：
- 在"具体元素"部分使用编号列表
- 确保每个可见元素都有独立的描述项
- 对复杂元素使用嵌套列表描述内部结构

**问题 5：生成的视觉风格不一致**

解决方案：
- 在第一个页面详细定义设计 token
- 后续页面在"视觉风格"部分引用第一个页面
- 明确列出具体色值、字号、字重、圆角、间距规范

**问题 6：交互状态页面关联不清**

解决方案：
- 在"交互说明"中明确写出前后关联的页面名称
- 描述触发条件时说明具体的操作和元素
- 描述后续转换时说明每种情况的具体结果

**问题 7：多页面没有在 Figma 上并排排列**

解决方案：
- 确保在提示词中明确列出每个页面的坐标（x、y）
- 使用绝对坐标而非相对位置描述
- 坐标计算：第一页 x=0，后续页面 x = 前一页 x + 前一页宽度 + 间距（通常 40px）
- 所有页面 y=0（从顶部开始）
- 在全局强制约束中明确说明页面坐标

**问题 8：多页面的视觉风格不一致**

解决方案：
