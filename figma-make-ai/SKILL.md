# Figma Make AI 提示词生成器（左-右布局）

---

## 使用说明

### 何时使用此 Skill

当用户需要以下内容时，使用此 Skill：
- 使用 Figma Make AI 生成 UI 设计稿
- 创建包含交互说明的设计文档
- 产品/设计/开发协作中的设计方案输出
- 替代传统标注稿，实现 AI 设计稿 + 交互文档合一

### 使用方式

#### 触发方式

用户可以通过以下方式触发此 Skill：
- 直接请求："帮我生成 Figma Make AI 提示词"、"用 Figma Make AI 设计这个页面"
- 描述需求："我需要为这个功能生成 UI 设计稿"
- 使用 Skill 命令：`/figma-make-ai`

#### 上下文要求

此 Skill 会从以下来源提取信息：
- **产品/功能需求**: 用户提供的任意形式需求描述
  - 一句话想法
  - PRD 摘要
  - 用户流程
  - 功能列表

## Skill 目标

当用户输入一个产品或功能需求时，自动完成以下任务：

1. 分析并结构化用户需求
2. 按「功能页面」进行拆分
3. 为每个功能模块生成：
   - 左侧：交互文档 Frame（包含所有页面的流程、状态、详细说明）
   - 右侧：所有页面垂直堆叠（主页面 + 所有交互状态页面）
4. 输出 **可直接用于 Figma Make AI 的中文提示词**
5. 每个交互状态作为独立完整页面，不是弹窗浮层

---

## 全局输出规则（必须严格遵守）

### 1. 输出格式

- 使用 Markdown
- 只使用纯文本 Markdown
- 不包含图片、链接、emoji
- 不做任何 Markdown 渲染说明
- 确保中文内容正确编码（UTF-8）

### 2. 语言

- 所有内容必须为中文
- 所有提示词必须是「直接对 Figma Make AI 说的话」

### 3. 页面拆分原则

- 每一个「用户可感知的界面」= 一个页面
- 每一个「交互状态」= 独立页面（不是弹窗浮层）
- 必须为以下状态创建独立页面：
  - 弹窗 / 抽屉内容
  - 二次确认界面
  - 成功提示页面
  - 失败错误页面
  - 加载状态页面
  - 空状态页面
- 所有相关页面在右侧垂直堆叠
- 左侧交互文档 Frame 包含所有页面的交互说明

### 4. 尺寸规则（必须明确写出）

- 移动端：宽度 375 px，高度不限制（自适应内容）
- 桌面端：宽度 1920 px，高度不限制（自适应内容）
- 每个页面都必须单独声明端类型与尺寸
- 最小高度建议：移动端 812 px、桌面端 1080 px
- 交互文档 Frame：宽度 600 px（移动端）/ 800 px（桌面端），高度自适应

### 5. 布局强制要求（方式二：左-右布局）

- 同一个 Figma Page 中，采用左-右布局：
  - **左侧**：交互文档 Frame（固定宽度 600-800px，高度自适应）
  - **右侧**：所有页面垂直堆叠（主页面 + 所有交互状态页面）
- 左右侧间距：150 px
- 右侧页面间距：100 px
- 每个交互状态都是独立的完整页面，不是浮层
- 交互文档 Frame 包含所有交互流程说明、页面跳转关系、状态列表

### 6. 禁止事项

- 不输出设计理念解释
- 不合并多个页面
- 不省略交互
- 不假设技术实现

### 7. 交互文档 Frame（左侧固定区域）

- 在所有页面的左侧，创建一个「交互文档」Frame
- 宽度建议：600 px（移动端）/ 800 px（桌面端）
- 高度：自适应内容，与右侧页面总高度对齐
- 内容结构：
  - 页面流程图（文字描述，使用箭头和缩进表示流程）
  - 所有交互状态列表（编号列表）
  - 每个交互的详细说明（触发位置、触发条件、展示页面、页面内容、用户操作）
- 格式：清晰的文本列表，层级分明，不需要视觉装饰

## Claude 内部执行流程（Skill 逻辑）

### 第一步：需求分析

- 识别：
  - 核心用户
  - 核心目标
  - 使用场景
- 拆解：
  - 功能模块
  - 页面清单

### 第二步：页面级拆分

- 每一个页面需明确：
  - 页面名称
  - 页面用途
  - 是否有附属交互（弹窗 / 状态）

### 第三步：生成 Figma Make AI 提示词（左-右布局）

为每个功能模块，生成完整提示词，要求 Figma Make AI：

1. 在画布左侧创建交互文档 Frame
2. 在画布右侧垂直堆叠所有页面（主页面 + 所有交互状态页面）
3. 左右间距 150px，右侧页面间距 100px

## 示例工作流

**用户请求**: "帮我为一个电商 APP 设计商品详情页"

**AI 应该执行**:

1. **需求分析**
   - 核心用户：网购消费者
   - 核心目标：了解商品信息并下单购买
   - 使用场景：浏览商品、查看详情、加入购物车

2. **页面拆分**
   - 页面组 1：商品详情
     - 页面 1.1：商品详情主页面
     - 页面 1.2：规格选择页面（交互状态）
     - 页面 1.3：加入购物车成功页面（交互状态）
   - 左侧：交互文档 Frame（包含所有交互流程说明）

3. **生成提示词**
   - 生成左侧交互文档 Frame 的提示词
   - 为右侧每个页面生成独立的 Figma Make AI 提示词
   - 所有页面垂直堆叠在同一画布右侧
