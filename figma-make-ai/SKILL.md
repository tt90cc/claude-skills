# Figma Make AI 提示词生成器 v3.0

## 核心特性

- 单页面生成，每个页面独立生成提示词
- 移动端 375×812px，pc端 1280×800px（固定高度，超出内容主区域滚动）
- 结合 /ui-ux-pro-max skill 生成设计指导
- 页面提示词按关联性紧密度排序（核心优先）

---

## 使用说明

**触发方式：** 直接请求、描述需求、或使用 `/figma-make-ai` 命令

**上下文：** 从用户提供的需求描述（一句话想法/PRD/用户流程/功能列表）中提取信息

---

## 全局规则

**输出格式：** 纯文本 Markdown，中文内容，不使用图片/链接/表格/emoji

**页面定义：** 每个交互状态都是一个独立页面（主功能页、弹窗、成功/失败提示、加载/空状态等）

**尺寸规范：**
- 移动端：375×812px（固定）
- pc端：1280×800px（固定）
- 滚动：主内容区域滚动（使用 min-h-0 + scrollbar-hide 方案隐藏滚动条），页面整体固定
- 自定义：用户明确提供的尺寸优先

**安全区规范（小程序/App端）：**
- 顶部安全区：状态栏区域（约44px），内容需避开或使用 safe-area-inset-top
- 底部安全区：导航栏/Home指示器区域（约34px），内容需避开或使用 safe-area-inset-bottom
- 胶囊按钮：右上角系统胶囊（约32×18px），页面布局需避免与之重叠
- 安全区约束需在【全局强制约束】中声明

**禁止事项：** 不输出设计理念、不省略交互、不假设技术实现、不使用相对宽度

## 执行流程

### Step 0：设计规范检查（前置必需）

**硬编码路径：** `docs/design-tokens.md`（相对于项目根目录）

1. 检查 `docs/design-tokens.md` 是否存在
2. **如果文件不存在：立即停止执行**，输出以下提示：

   ```
   ⚠️ 设计规范文件不存在

   生成 Figma Make AI 提示词前，必须先创建项目设计规范。

   请执行以下步骤：
   1. 在项目根目录创建 docs/design-tokens.md
   2. 定义项目的颜色、圆角、间距、阴影等设计规范（命名不限制）
   3. 设计规范创建完成后，再重新执行此任务
   ```

3. **如果文件存在：** 读取完整内容，在后续所有步骤中严格遵循其中的规范
4. **动态提取原则：** 从 docs/design-tokens.md 中读取所有内容，识别并提取：
   - 颜色定义（查找颜色相关表格或列表，提取变量名和十六进制值）
   - 圆角定义（查找 radius/border-radius 相关值）
   - 间距定义（查找 space/spacing 相关值）
   - 阴影定义（查找 shadow/elevation 相关值）
   - 布局原则（查找布局/留白/边框相关规则）
   - 如设计规范使用其他命名方式，按实际内容提取
5. 将提取的设计规范作为【设计规范约束】添加到每个提示词的【全局强制约束】之前

### Step 1：需求理解与确认

需求理解与确认 → 确定尺寸 → 列出页面清单 → 与用户确认

### Step 2：调用 /ui-ux-pro-max 获取设计风格指导

调用 /ui-ux-pro-max 获取设计风格指导，结合 design-tokens.md 中的规范生成统一的设计指导

### Step 3：按关联性紧密度排序

按关联性紧密度排序：核心功能页 → 子功能页 → 交互状态页 → 辅助状态页

### Step 4：为每个页面独立生成 Figma Make AI 提示词

为每个页面独立生成 Figma Make AI 提示词，**严格遵循 design-tokens.md 中的设计规范**

### Step 5：验证生成结果

验证生成结果（尺寸、滚动规则、强制约束声明、设计规范遵循性）

#### Step 5.1：尺寸与约束验证
- [ ] 画板宽度是否为 375px（移动端）或 1280px（pc端）
- [ ] 画板高度是否固定
- [ ] 滚动规则是否声明
- [ ] 【全局强制约束】是否完整

#### Step 5.2：设计规范合规性逐项检查

**必须严格对照 `docs/design-tokens.md` 进行每一项检查：**

1. **配色方案检查**
   - [ ] 所有色值是否来自 design-tokens.md 中定义的颜色
   - [ ] 是否引入了 design-tokens.md 中不存在的色系（如蓝色渐变、红色等）
   - [ ] 十六进制色值是否与 design-tokens.md 完全一致

2. **圆角规范检查**
   - [ ] 容器圆角是否符合设计规范中定义的值
   - [ ] 组件圆角是否符合设计规范中定义的值
   - [ ] 胶囊按钮圆角是否符合设计规范中定义的值

3. **间距规范检查**
   - [ ] 左右安全边距是否符合设计规范中定义的值
   - [ ] 卡片内部 Padding 是否符合设计规范中定义的值

4. **阴影规范检查**
   - [ ] 是否仅使用设计规范中定义的阴影值
   - [ ] 是否有元素使用了未经定义的其他阴影效果

5. **渐变逻辑检查**
   - [ ] Header 区域渐变起点是否符合设计规范中定义的主色
   - [ ] 渐变终点是否为设计规范中定义的页面背景色
   - [ ] 是否存在未经 design-tokens.md 定义的渐变色系

6. **布局原则检查**
   - [ ] 是否遵循设计规范中定义的布局原则
   - [ ] 是否使用了过细的分割线

#### Step 5.3：元素样式逐项核对

对照 design-tokens.md，逐个验证提示词中每个元素的颜色是否合规：
- 按钮背景色是否来自设计规范中定义的主色或 accent
- 输入框背景色是否为设计规范中定义的背景色或卡片色
- 文本颜色是否为设计规范中定义的文本色
- 任何新引入的颜色必须先确认 design-tokens.md 中是否存在

#### Step 5.4：发现问题则重新生成

验证过程中发现任何不合规项，必须：
1. 记录不合规的颜色/样式
2. 从 design-tokens.md 找到合规的替代方案
3. 重新生成该页面的提示词
4. 再次执行 Step 5 的全部检查项

**重要提醒：**
- 原型代码中的颜色不等于合规颜色
- design-tokens.md 是唯一的设计规范来源
- ui-ux-pro-max 返回的推荐色系仅作参考，必须经 design-tokens.md 验证后才可使用

## 设计规范应用

### 强制遵循原则

从 `docs/design-tokens.md` 读取设计规范后，必须在每个提示词中严格应用：

1. **配色方案**：必须使用定义好的 HSL 变量/十六进制色值，**严禁引入未经定义的其他色系**
2. **圆角规范**：所有容器圆角必须按照设计规范中定义的 radius 值使用（具体值从文件读取）
3. **布局原则**：遵循设计规范中定义的布局原则
4. **阴影规范**：仅使用设计规范中定义的阴影值（具体值从文件读取）
5. **间距规范**：按照设计规范中定义的间距值使用（具体值从文件读取）
6. **渐变逻辑**：Header 区域渐变按照设计规范中定义的方式（从主色向下过渡到页面背景色）

### 提示词中的声明方式

在每个页面提示词的【全局强制约束】之前添加【设计规范约束】：

```
**【设计规范约束】**（来源：docs/design-tokens.md）
- 配色：[从 docs/design-tokens.md 动态提取所有颜色定义]
- 圆角：[从 docs/design-tokens.md 动态提取圆角规范]
- 间距：[从 docs/design-tokens.md 动态提取间距规范]
- 阴影：[从 docs/design-tokens.md 动态提取阴影规范]
- 布局：[从 docs/design-tokens.md 动态提取布局原则]

**【全局强制约束】**
[原有约束...]
```

### 设计一致性

在第一个页面详细定义视觉风格规范（来自 design-tokens.md），后续页面引用并复用这些规范。使用 /ui-ux-pro-max 生成统一的设计指导，但所有设计决策必须与 design-tokens.md 一致。

## 输出要求

- 纯文本 Markdown，不使用图片/链接/表格/emoji
- UTF-8 编码，全中文内容
- 遵循标准化提示词模板，必填字段不省略
- 禁止：设计理论讨论、多种方案选择、技术实现假设、模糊描述
- 必需：明确尺寸、十六进制色值、带单位 px 的数值、完整元素列表、清晰交互说明

## 完成标准

- 尺寸：移动端 375×812px，pc端 1280×800px，固定高度

- 风格：视觉规范完整，多页面一致（通过 /ui-ux-pro-max 统一）
- 格式：中文纯文本 Markdown，无图片/链接/emoji，十六进制色值，带单位 px
