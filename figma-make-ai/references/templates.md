## 完整示例：订单删除确认页面

**用户需求：** "帮我设计一个移动端的订单删除确认页面。"

**Figma Make AI 提示词：**

**【设计规范约束】**（来源：docs/design-tokens.md）
- 配色：[从 docs/design-tokens.md 动态提取所有颜色定义]
- 圆角：[从 docs/design-tokens.md 动态提取圆角规范]
- 间距：[从 docs/design-tokens.md 动态提取间距规范]
- 阴影：[从 docs/design-tokens.md 动态提取阴影规范]
- 布局：[从 docs/design-tokens.md 动态提取布局原则]

**【全局强制约束】**
- 画板宽度严格等于 375px，不得自动调整
- 画板高度固定为 812px，不得自动调整
- 滚动规则：主内容区域高度超过固定高度时，主内容区域内部滚动展示，页面整体高度固定不变
- 导航栏和底部Tab栏固定定位，不随内容滚动
- 所有元素宽度不得超过 375px

**页面类型：** 移动端
**尺寸：** 375 × 812 px（固定高度，超出内容主区域滚动）

**页面描述：**
用户删除订单前的二次确认弹窗页面，防止用户误删订单。

**布局结构：**
- 底层：半透明遮罩层，覆盖整个屏幕，[从设计规范提取透明度]
- 中央：确认弹窗容器，位于屏幕垂直和水平居中

**具体元素：**

**弹窗容器：**
- 类型：卡片背景矩形容器
- 尺寸：宽度 310px，高度自适应内容
- 样式：[从设计规范提取背景色、圆角、阴影]
- 位置：屏幕居中

**弹窗内部元素（从上到下）：**

1. **标题区域**（距容器顶部 [从设计规范提取间距]）
   - 类型：文本标题
   - 内容："确认删除订单？"
   - 样式：字号 18px，字重 600，[从设计规范提取文本色]，居中对齐

2. **提示文案**（距标题 [从设计规范提取间距]）
   - 类型：辅助说明文本
   - 内容："订单删除后将无法恢复"
   - 样式：字号 14px，字重 400，[从设计规范提取辅助文本色]，居中对齐

3. **订单信息区域**（距提示文案 [从设计规范提取间距]）
   - 类型：信息展示容器
   - 样式：[从设计规范提取背景色和圆角]，[从设计规范提取内边距]

   内部元素：
   - 订单号文本："订单号：202401280001"，字号 13px，[从设计规范提取文本色]
   - 金额文本："金额：¥299.00"，字号 13px，[从设计规范提取文本色]

4. **按钮区域**
   - 取消按钮：[从设计规范提取次要按钮样式]，文字"取消"
   - 确认删除按钮：[从设计规范提取危险/警告按钮样式]，文字"确认删除"

**交互说明：**

**触发条件：**
- 来源页面：订单列表页
- 触发操作：点击某个订单项右侧的删除图标

**后续转换：**
1. 点击"取消"按钮 → 关闭弹窗，返回订单列表页
2. 点击"确认删除"按钮 → 删除成功/失败页面

**视觉风格：**

## 页面类型模板

### 弹窗/对话框类型页面
```
**布局结构：**
- 底层：半透明遮罩，覆盖整个屏幕，[从设计规范提取遮罩透明度]
- 中央：弹窗容器，[尺寸]，[从设计规范提取圆角]，[从设计规范提取阴影]

**关键要点：**
- 明确遮罩层的透明度
- 说明弹窗的对齐方式（居中/底部/顶部）
- 描述弹窗的进入/退出动画
```

### 表单页面
```
**关键要点：**
- 列出所有表单字段及其类型
- 说明必填/选填标识
- 描述输入验证规则和错误提示样式
- 说明提交按钮的禁用/可用状态
```

### 列表页面
```
**关键要点：**
- 描述列表项的结构和内容
- 说明列表项之间的间距
- 描述滚动行为（是否有上拉加载、下拉刷新）
- 说明空状态的展示
```

### 加载/空状态页面
```
**关键要点：**
- 描述加载指示器的样式和位置
- 说明加载文案（如"加载中..."）
- 空状态需要包含图标/插画、说明文字、操作按钮（如有）
```

## 尺寸预设

### 移动端
- 宽度：375 px（固定）
- 高度：812 px（固定）
- 适用：手机App、移动网页

### 桌面端
- 宽度：1280 px（固定）
- 高度：800 px（固定）
- 适用：桌面网页、后台管理系统

### 自定义尺寸
- 格式：[宽度] × [高度] px
- 示例：414 × 896 px（iPhone 11 屏幕）
