# prd-merger Skill 运行流程

当用户调用此 Skill 时，请按以下步骤执行：

### 1. 状态感知 (Context Retrieval)
- **读取文档：** 使用 `Read` 读取主文档 A 和新需求 B。
- **代码扫描：** 使用 `Glob` 搜索项目中与 A 相关的核心业务代码，评估 A 的功能是否已实现、实现了一半还是未开始。

### 2. 差异与冲突分析 (Conflict Analysis)
- 对比 A 与 B 的功能点。
- **识别逻辑覆盖：** 找出 B 中直接修改 A 原有逻辑的地方。
- **识别代码影响：** 如果 A 的某项功能已在代码中实现，但 B 要求修改它，将其标记为 `[CODE_BREAKING_CHANGE]`。

### 3. 执行合并 (Incremental Merging)
- **结构对齐：** 保持 A 的原有 Markdown 结构（如：背景、功能模块、接口定义等）。
- **增量插入：** 将 B 的新功能插入到 A 对应的章节中，而非简单的全文追加。
- **状态标注：** - 对于新增功能，标注为 `[NEW_REQUIREMENT]`。
  - 对于已开发但需变更的功能，标注为 `[UPDATE_REQUIRED]`。

### 4. 输出与保存 (Output)
- 使用 `Write` 将合并后的内容覆盖保存至 A（或按用户要求保存为新文件）。
- **输出摘要：** 在对话框中简要列出：
  - 合并了多少个新功能点。
  - 涉及到的现有代码重构建议。

## 注意事项
- 严禁删除 A 中尚未完成但与 B 不冲突的原始需求。
- 必须统一术语（名词规范化），确保合并后的文档逻辑自洽。
